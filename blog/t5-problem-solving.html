That time i got blocked on a simple problem and that time I solved a problem in an elegant way

Often times the only way for me to solve a problem in an elegant way is to first solve it in a non-elegant way when it comes to code. One time I was solving a coding problem and was delighted when it worked only to realise later how bad my code was and cleaning it up felt good and elegant. The techniques I use when in trouble is to separate components as much as possible and try to get simple incremental parts to work. Also I always have pen and paper lying around handy to write and draw when needed. Hand-writing and sketching really does help me.

Problem solving techniques:
Pseudocode

Pseudocode is a great tool to think about the future solution. Sometimes written out, sometimes just said outloud while typing code.

Trying something

Trying things is the best thing ever and even if the thing you are trying is not working, the chances are you learnt something while trying.

Rubber ducky method

Explaining the problem and the approaches you took to solve it and why they failed and how are a powerful method to problem solve. If you use a rubber ducky you don't have to take up other people's time.

Reading error messages

Error messages are great. I read them and sometimes they help. Other times they don't.

Console.logging

console.logging is pretty good to keep an eye on a variable or something.

Googling

I always google and sometimes it helps.

Asking your peers for help

This is my favourite, because you can ask real human questions if something doesn't quite make sense ton you and it can save you a lot of time.

Asking coaches for help

I haven't asked coaches for help yet. But I imagine it would be the same as peers but more experienced replies. At any rate asking real human for help is very effective and I like doing it because you can learn and solve much faster.

Improving your process with reflection

What worked and what didn't are the questions that drive improvement of every sprint. Reflection is a requirement and does help with the next sprint and understanding yourself and own dynamic.

JS Array methods

.map()
method is an array method. What it does is apply some sort of (specified) action to an existing array's elements and produce a new array based on the action applied to every single element of the existing array.

.filter()
method is an array method. It applies a function to every element of the original array and returns only those elements that return the value 'true' as a result of the function application.

.reduce()
method is an array method. What it does is turn an array into a single value. The value is an operation performed on the elements of an array that you specify yourself.
